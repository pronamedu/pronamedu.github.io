8. what are the advanteges and disandvanges of using DHCP surver​


Answer:

advantages of DHCP:

➨DHCP is easy to implement and does automatic assignment of IP addresses to requesting clients. Hence manual configuration time of IP addresses can be reduced.

➨The implementation does not require any additional costs.

➨Duplicate or invalid assignment of IP addresses are prevented. Hence there is no chance of conflicts in IP addresses.

➨It simplifies administration of the network.

➨It supports multiple scopes e.g. multicast scope, super scope etc.

➨It has great benefit to mobile users as valid configuration parameters are automatically obtained from the new network.

Explanation:

disadvantages of DHCP:

➨DHCP server can be single point of failure in networks having only one configured DHCP server.

➨DHCP packets can not travel across router, Hence relay agent is necessary to have DHCP server handle all leases on both network segments. Relay agents receive broadcast DHCP packets and forward them as unicast packets to DHCP server. Here relay agent must be configured with IP address of the DHCP server.

➨Security: As DHCP server has no secure mechanism for authentication of the client, it can gain unauthorized access to IP addresses by presenting credentials such as client identifiers which belong to other DHCP clients.

➨The machine name does not change when new IP address is assigned.

➨Client is not able to access the network in the absence of the DHCP server.

15. write the procedure in installing DHCP server role and configure ​


Answer:

Sure, here's a general procedure for installing and configuring a DHCP server role on a Windows Server:

1. Open the Server Manager on your Windows Server.

2. Click on the "Add Roles and Features" option, which will start the installation wizard.

3. Select "Role-based or feature-based installation" and click "Next".

4. Choose the server you want to install the DHCP role on and click "Next".

5. Select the "DHCP Server" role from the list of available roles and click "Next".

6. Review the information on the "DHCP Server" screen and click "Next".

7. Click "Install" to begin the installation process.

8. Once the installation is complete, click "Close".

Now that the DHCP server role is installed, you can configure it by following these steps:

1. Open the DHCP management console from the Server Manager.

2. Right-click on the server name in the console tree and select "Authorize" to allow the DHCP server to operate in the Active Directory domain.

3. Right-click on the "IPv4" node in the console tree and select "New Scope" to create a new scope.

4. Follow the wizard to configure the scope, including the IP address range, subnet mask, default gateway, DNS server, and lease duration.

5. Once the scope is configured, right-click on it and select "Activate" to make it available to clients.

6. You can also configure additional DHCP options, such as DNS suffixes and WINS servers, by right-clicking on the server name and selecting "Set Predefined Options".

7. Finally, you can configure DHCP reservations for specific clients by right-clicking on the "Reservations" node and selecting "New Reservation". Here, you can specify the MAC address and IP address of the client.

That's it! Your DHCP server is now configured and ready to provide IP addresses to clients on your network.

How to Verify If DHCP is Enabled

Using Command Prompt on Windows

Using a command on Windows, we can verify that a working computer is setup via DHCP instead of Static IP.

On a Windows computer, have the customer click on Start, then select Run. If they don't see Run, they can get to it by pressing the Windows key+R.

Type in cmd and hit Ok.

When the command prompt window appears, type in ipconfig /all and hit Enter.

Look for the setting Dhcp Enabled. This should be set to Yes.
